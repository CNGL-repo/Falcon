Globic Log Service - Example of Generated Provenance Data and Retrieval

Generated Provenance Data Example

The following is an example of generated provenance data by the service.
Assuming the following request is made to the service:

POST http://www.example.com/globicLog/service/MockComponent_1/?password=123456&activity=my_translate_92&user=Joe_Bloggs&contentConsumed1=<http://www.example.com/filestore/document_1>&contentConsumed1Type=nif:String&&contentConsumed2=<http://www.example.com/filestore/document_2>&contentConsumed2Type=nif:Word&contentGenerated1=<http://www.example.com/filestore/document_3>&contentGenerated1Type=gic:Translation&contentGenerated2=<http://www.example.com/filestore/document_4>&contentGenerated2Type=gic:Translation

The following provenance data is generated by the service and stored in the triple-store:

gic:MockComponent_1 rdf:type gic:ContentProcessingComponent
gic:Joe_Bloggs rdf:type prov:Person
<http://www.example.com/filestore/document_1> rdf:type nif:String
<http://www.example.com/filestore/document_2> rdf:type nif:Word
gic:my_translate_92 rdf:type gic:Translate
gic:my_translate_92 prov:wasAssociatedWith gic:MockComponent_1
gic:my_translate_92 prov:wasAssociatedWith gic:Joe_Bloggs
gic:my_translate_92 prov:startedAtTime "2013-09-12T11:08:09+01:00"^^xsd:dateTime
gic:my_translate_92 gic:contentConsumed <http://www.example.com/filestore/document_1>
gic:my_translate_92 gic:contentConsumed <http://www.example.com/filestore/document_2>
<http://www.example.com/filestore/document_3> rdf:type gic:Translation
<http://www.example.com/filestore/document_3> prov:wasDerivedFrom <http://www.example.com/filestore/document_1>
<http://www.example.com/filestore/document_3> prov:wasDerivedFrom <http://www.example.com/filestore/document_2>
<http://www.example.com/filestore/document_3> prov:generatedAtTime "2013-09-12T11:08:09+01:00"^^xsd:dateTime
<http://www.example.com/filestore/document_3> gic:contentGeneratedBy gic:my_translate_92
<http://www.example.com/filestore/document_4> rdf:type gic:Translation
<http://www.example.com/filestore/document_4> prov:wasDerivedFrom <http://www.example.com/filestore/document_1>
<http://www.example.com/filestore/document_4> prov:wasDerivedFrom <http://www.example.com/filestore/document_2>
<http://www.example.com/filestore/document_4> prov:generatedAtTime "2013-09-12T11:08:09+01:00"^^xsd:dateTime
<http://www.example.com/filestore/document_4> gic:contentGeneratedBy gic:my_translate_92

Retrieval of Provenance Data Example

The following are examples of retrieval of provenance data from the service.
Note that the data returned by the service is in JSON format, the examples below show the data in decoded form and in tabular format.

  • Request to retrieve provenance data about a piece of content.
    Assuming the following request is made to the service:

    GET http://www.example.com/globicLog/service/MockComponent_1/?password=123456&retrieveData=<http://www.example.com/filestore/document_3>

    The following provenance data about "document_3" is returned by the service (decoded from JSON and put in table):

    • http://www.w3.org/1999/02/22-rdf-syntax-ns#type http://www.cngl.ie/ontologies/gicup#Translation
    http://www.w3.org/ns/prov#wasDerivedFrom http://www.example.com/filestore/document_1
    http://www.w3.org/ns/prov#wasDerivedFrom http://www.example.com/filestore/document_2
    http://www.w3.org/ns/prov#generatedAtTime "2013-09-12T11:08:09+1:00"^^xsd:dateTime
    http://www.cngl.ie/ontologies/gicup#contentGeneratedBy http://www.cngl.ie/ontologies/gicup#my_translate_92


  • Request to retrieve provenance data about a component.
    Assuming the following request is made to the service:

    GET http://www.example.com/globicLog/service/MockComponent_1/?password=123456&retrieveData=gic:MockComponent_1

    The following provenance data about "MockComponent_1" is returned by the service (decoded from JSON and put in table):

    • http://www.w3.org/1999/02/22-rdf-syntax-ns#type http://www.cngl.ie/ontologies/gicup#contentProcessingComponent


  • Request to retrieve provenance data about an activity.
    Assuming the following request is made to the service:

    GET http://www.example.com/globicLog/service/MockComponent_1/?password=123456&retrieveData=gic:my_translate_92

    The following provenance data about "my_translate_92" is returned by the service (decoded from JSON and put in table):

    • http://www.w3.org/1999/02/22-rdf-syntax-ns#type http://www.cngl.ie/ontologies/gicup#Translate
    http://www.w3.org/ns/prov#wasAssociatedWith http://www.cngl.ie/ontologies/gicup#MockComponent_1
    http://www.w3.org/ns/prov#wasAssociatedWith http://www.cngl.ie/ontologies/gicup#Joe_Bloggs
    http://www.w3.org/ns/prov#startedAtTime "2013-09-12T11:08:09+1:00"^^xsd:dateTime
    http://www.cngl.ie/ontologies/gicup#contentConsumed http://www.example.com/filestore/document_1
    http://www.cngl.ie/ontologies/gicup#contentConsumed http://www.example.com/filestore/document_2


  • Request to retrieve provenance data about a user.
    Assuming the following request is made to the service:

    GET http://www.example.com/globicLog/service/MockComponent_1/?password=123456&retrieveData=gic:Joe_Bloggs

    The following provenance data about "Joe_Bloggs" is returned by the service (decoded from JSON and put in table):

    • http://www.w3.org/1999/02/22-rdf-syntax-ns#type http://www.w3.org/ns/prov#Person
